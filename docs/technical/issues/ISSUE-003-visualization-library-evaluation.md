# Enhancement: Оцінити сучасний стек візуалізацій для енергетичного dashboard

## Problem

Для нового dashboard потрібні сильніші графіки, ніж у legacy варіанті. Поки що не зафіксовано, чи використовувати D3, високорівневу modern chart library, чи гібридний підхід. Без короткого evaluation легко застрягти між слабким boilerplate charting і занадто важким low-level D3 everywhere.

## Research Questions

- Яка бібліотека найкраще підходить для щільних time-series і energy/market analytics views у Nuxt?
- Чи варто брати high-level chart system для 80% графіків і D3 лише для signature visuals?
- Які графіки дадуть найкращий ефект для:
  - DAM historical vs forecast overlay
  - dispatch plan ladder / schedule ribbon
  - SOC / SOH operational state
  - regret vs oracle comparison
  - weather/solar contribution context

## Scope

- Зробити короткий benchmark кількох варіантів:
  - D3
  - Observable Plot
  - Apache ECharts
  - Visx
  - будь-яка доречна сучасна library для rich analytical dashboards
- Оцінити критерії:
  - visual expressiveness
  - integration friction with Nuxt
  - performance for multi-series time data
  - ease of custom branded visuals
  - maintainability for AI-assisted codebase work
- Запропонувати рекомендований стек і fallback strategy.

## Acceptance Criteria

- Є коротка decision note з порівнянням мінімум 3 варіантів.
- Є список рекомендованих chart patterns для MVP dashboard.
- Є рішення, де достатньо library-level charts, а де потрібен custom D3 layer.
